NAME

xrtph1sim -- Converts ASCA FITS files to Swift XRT Photon Counting Events FITS files.

USAGE

xrtph1sim infile outfile dateobs timeobs

DESCRIPTION

The XRT Photon Counting mode maps onto the ASCA SIS mode called Faint. This data collecting mode returns the PHA value of a 3x3 box around each event. We decided to build a Level1 event file of Swift XRT, for software developing purposes, using the data collected by ASCA SIS. This task creates an XRT Photon Counting events FITS file containing an EVENTS binary table extension and a GTI binary table extension by reading ASCA Faint events FITS file. Some columns present in the ASCA input file will be processed and written to the output file.

PARAMETERS

infile [asca file name]
The name of the ASCA Faint events FITS file to be processed and converted into the XRT Photon counting events file.

outfile [output file name]
The name of the output file. The FITS file will have an empty primary array followed by two extensions.

dateobs [string]
Observation start date. The format must be: yyyy-mm-dd

timeobs [string]
Observation start time. The format must be hh:mm:ss

(clobber = no) [boolean]
If true, an existing file with the same name as the specified output file will be overwritten.

(chatter = 2) [integer]
Verbosity level, from 0 to 5.

(history = yes) [boolean]
Write parameter values in HISTORY block.

EXAMPLES

  1. Create a FITS file called 'xrtph1out.fits' from an ASCA file called 'ascafile.fits'. 

    ft> xrtph1sim ascafile.fits xrtphout.fits clobber=no verbosity=5
    The command line above unset clobber and set the maximum verbosity level.
